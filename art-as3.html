Flash Player は 2020年12月31日にサポート終了

ActionScript 3.0 は Flash Player / Adobe AIR 上で動作します。現在のブラウザは Flash を内蔵していないため、コンパイルして Adobe Flash Player Standalone（スタンドアロン版）で実行する必要があります。
ソースファイル as3/InteractiveArt.as を以下の手順でコンパイルしてください。

SWF 実行エリア

コンパイル済みの as3/InteractiveArt.swf があれば、Adobe Flash Player Standalone で直接開いてください。Ruffle（WebAssembly 版 Flash エミュレータ）による試験的な実行も可能ですが、Ruffle はカメラ API を未サポートのため映像入力は動作しません。

🎬

SWF ファイルが必要です

as3/InteractiveArt.as を Apache Flex SDK または Adobe Animate CC でコンパイルし、生成された as3/InteractiveArt.swf を Adobe Flash Player Standalone で実行してください。

コンパイルと実行の手順

1

Apache Flex SDK をダウンロード（無料）

flex.apache.org/installer.html から Apache Flex SDK インストーラーをダウンロードして展開する。

2

mxmlc コマンドでコンパイル

コマンドプロンプトまたは PowerShell で以下を実行：

COMMAND PROMPT
cd C:\Users\kirin\interactive-art\as3
C:\flex-sdk\bin\mxmlc.exe InteractiveArt.as -output InteractiveArt.swf
3

Adobe Flash Player Standalone をダウンロード

Adobe Debug Downloads から「Flash Player projector」（Windows 版 .exe）をダウンロード。インストール不要、単体実行可能。

4

SWF を Flash Player で実行

Flash Player Standalone を起動し、「ファイル → 開く」から InteractiveArt.swf を選択。カメラアクセスの許可を求めるダイアログが表示されたら「許可」を選ぶ。

💡
Adobe Animate CC を使う場合：新規 ActionScript 3.0 ドキュメントを作成 → ドキュメントクラスに InteractiveArt を指定 → InteractiveArt.as を同フォルダに置いて「ムービープレビュー（Ctrl+Enter）」でコンパイル実行。

BitmapData.compare() の動作詳細

これが論文で言及されていた「ActionScript の関数」の正体です。

前フレーム (BitmapData) prevBD
→
ActionScript 組み込み prevBD.compare(curBD)
→
差分ビットマップ diffBD
→
閾値判定 motionPoints[]

compare() の戻り値

戻り値意味
0（整数）2フレームが完全に同一 → 動体なし
-3（整数）幅が異なる（通常発生しない）
-4（整数）高さが異なる（通常発生しない）
BitmapData差分ビットマップ（各ピクセルが差分値）

差分ビットマップのピクセル形式（ARGB 32bit）

alpha ビットRGB ビット意味
0x000x000000前後フレームで同一ピクセル → 動体なし
0xFFR=|ΔR|, G=|ΔG|, B=|ΔB|差異あり → (R+G+B)/3 > threshold なら採用
