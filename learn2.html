共通基盤 — 画像差分法の数学
ピクセルの「引き算」で動きを検出する仕組み
両作品共通

コンピュータにとって、映像は「ピクセル（画素）」の集合です。各ピクセルは R（赤）・G（緑）・B（青） の3つの数値（0〜255）で色を表します。

例えば「純粋な赤」は R=255, G=0, B=0 と表現されます。

RGB カラーモデルとフレーム差分の仕組み
前フレームと現フレームの差分を計算 → 大きな差＝動体あり

画像差分法の計算式：各ピクセルで「今のフレーム」と「1フレーム前」の色の差を計算します。

差分値 = ( |R現在 − R前| + |G現在 − G前| + |B現在 − B前| ) ÷ 3

この差分値が 閾値（threshold） を超えたピクセル座標に、Evoloop のセルや Komorebi の光パッチを生成します。

閾値を小さくすると敏感（ちょっとした動きも検出）、大きくすると鈍感（大きな動きしか検出しない）になります。

// 画像差分法の核心コード (JavaScript) const step = 8; // 8ピクセルごとにサンプリング（高速化） for (let y = 0; y < H; y += step) { for (let x = 0; x < W; x += step) { const i = (y * W + x) * 4; // RGBA配列のインデックス const diff = ( Math.abs(cur[i] - prev[i]) + // R の差 Math.abs(cur[i+1] - prev[i+1]) + // G の差 Math.abs(cur[i+2] - prev[i+2]) // B の差 ) / 3; if (diff > threshold) { // 閾値を超えたら動体あり motionPoints.push({ x, y }); // 座標を記録 } } }

人工生命 Evoloop — セルラオートマタとは
「生きている表面」論文のコアとなる人工生命モデル
論文①「生きている表面」

セルラオートマタ（Cellular Automata）とは、格子状のセル（細胞）が単純なルールで状態を変化させ、複雑なパターンを生み出すモデルです。有名な例に「ライフゲーム（Game of Life）」があります。

論文で使われた Evoloop は、電気通信大学の佐山弘樹氏が開発した9状態・5近傍のセルラオートマタです。自己複製・衝突による消滅・進化という性質を持ちます。

自己複製（Self-Replication）

個体が周囲の空きスペースに自分のコピーを作る。空間が十分にあれば増殖し続ける。

衝突→消滅（Collision）

2つの個体が同じ場所に増殖しようとすると衝突が起き、一方または両方が消滅する。

進化（Evolution）

厳しい環境（スペース不足）でも生き残れるよう、個体が少しずつ小さく・強くなっていく。

空間の種類

空間A（動体エリア）：密集 → 高頻度衝突 → 激しい明滅
空間B（静止エリア）：まばら → ゆっくり進化

簡易セルラオートマタ — セルの誕生・複製・衝突・消滅
クリックでセルを追加できます
インタラクションの仕組み： カメラ前で人が動く → 動体検出エリア（空間A）に多数のセルが播種 → 密集 → 高頻度衝突 → 激しい明滅パターン。 動きを止めると → セルの密度が下がり → ゆっくりと進化（空間B）。

木漏れ日と光の物理 — Calm Technology
Komorebi 論文が着目した自然現象の科学
論文②「Komorebi」

木漏れ日（Komorebi）とは、木の葉の間から差し込む太陽の光のことです。日本語特有の言葉で、英語には対応する単語がありません。

論文はこの木漏れ日を Calm Technology（穏やかな技術） の表現媒体として使います。Calm Technology とは、人の注意を強制的に奪わず、周辺視野でそっと情報を伝えるデザイン思想です。

Calm Technology の原則

  • 注意を強制しない
  • 周辺視野に溶け込む
  • 静けさと時間の流れを感じさせる
  • 「余白（yohaku）」を持つ

木漏れ日がなぜ適しているか

  • 光のパターンが自然に変化
  • 周辺視野で認識できる
  • 穏やかさ・自然との一体感
  • 強く注意を引かない

光の物理：Hard Shadow vs Soft Shadow

論文では光源の種類によって影の硬さが変わることを活用しています。

点光源（Hard Shadow）

輪郭くっきり・本影(umbra)のみ

面光源（Soft Shadow）

縁がぼやける・半影(penumbra)あり

① 放射状グラデーション（Radial Gradient） — 木漏れ日パッチの描画

// hard patch: 小さく輪郭くっきり（点光源） const grad = ctx.createRadialGradient( 0, 0, 0, // 内側円（中心） 0, 0, r // 外側円（半径r） ); grad.addColorStop(0, `hsla(65,100%,80%,${a})`); grad.addColorStop(0.65, `hsla(65,100%,70%,${a*0.5})`); grad.addColorStop(1, `hsla(65,100%,80%,0)`); // soft patch: 広くぼんやり（面光源） grad.addColorStop(0, `hsla(65,90%,80%,${a})`); grad.addColorStop(0.35, `hsla(65,90%,75%,${a*0.6})`); grad.addColorStop(0.7, `hsla(65,90%,65%,${a*0.2})`); grad.addColorStop(1, `hsla(65,90%,80%,0)`);

② 加算合成（Additive Blending） — 光が重なって明るくなる

// 通常合成: 後から描いた色が上書き（重なっても明るくならない） ctx.globalCompositeOperation = 'source-over'; // 加算合成: RGBの値を加算 → 重なるほど明るくなる（現実の光と同じ） ctx.globalCompositeOperation = 'lighter'; // 例: RGB(100,50,0) + RGB(50,100,0) = RGB(150,150,0) → 黄色に！
通常合成 vs 加算合成（lighter）の違い

③ ベルの音（Web Audio API） — Calm Technology の音響

// 穏やかなベル音: ゆっくり立ち上がり、長く残響する const osc = audioCtx.createOscillator(); osc.type = 'sine'; osc.frequency.value = 700 - (y / H) * 350; // 上部=高音 const gain = audioCtx.createGain(); gain.gain.setValueAtTime(0, now); gain.gain.linearRampToValueAtTime(0.035, now + 0.03); // 遅いアタック gain.gain.exponentialRampToValueAtTime(0.001, now + 1.6); // 長い余韻 // 倍音を加えてベルらしくする const osc2 = audioCtx.createOscillator(); osc2.frequency.value = freq * 2.76; // 2.76倍音（鐘の特徴的な倍音比）
ポイント：Calm Technology の音は「気づかせるが邪魔しない」ことが重要。ゆっくりアタック・長い余韻で自然に溶け込む音を設計しています。500ms以上の間隔を設けて頻繁に鳴らさないことも大切。
