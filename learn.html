今回作ったのは、インタラクティブアートと呼ばれる作品です。

「インタラクティブ」とは「対話的な」という意味。つまり、見ている人が動くと映像がリアルタイムで変化する、「観客が参加できるアート」のことです。

身近な例えで言うと…
ゲームで自分のキャラクターを動かすと画面が変わりますよね？それと同じように、カメラの前で手を振ったり歩いたりすると、プロジェクターに映る映像がリアルタイムで変化します。

iPhoneのカメラ映像 → PCで処理 → プロジェクターで投影

カメラの映像って何でできてる？
フレームとピクセルを理解しよう

動画は実は、パラパラ漫画と同じです。

1秒間に約30枚の「静止画像」を連続して表示することで、映像が動いているように見えます。この1枚1枚の静止画を フレーム と呼びます。

そしてフレームの1枚は、細かい色の点（ピクセル）が集まって作られています。例えば1280×720サイズの画像は 921,600個 のピクセルで構成されています。

1秒間に約30枚のフレームが流れる → 動いているように見える

ピクセルの色は R・G・B の3つの数字で表現されます。
例えば赤色 → R:255, G:0, B:0　／　青色 → R:0, G:0, B:255
この数字を比べることで「色が変わったか」を判定できます。

動きをどうやって検出する？
画像差分法（がぞうさぶんほう）

動いている場所を見つけるための技術が 画像差分法 です。

考え方はとてもシンプルです。

11つ前のフレーム（静止画）を保存しておく
↓
2現在のフレームを取得する
↓
32枚を同じ座標のピクセルごとに比較する（引き算）
↓
4色の差が大きい場所 = 「動きがあった」と判定
前のフレーム
－
今のフレーム
＝
差分（動いた部分が光る）

※「閾値（しきいち）」= 「どのくらいの差なら反応するか」の設定値。調整できます。

例えて言うと…
朝と夜に同じ場所を撮った写真を重ねると、変わった部分（人が入った・出た）が浮き上がります。これが画像差分法の考え方です。

「座標」とは？
画面の左上が (x=0, y=0)、右に行くほどxが増え、下に行くほどyが増えます。動きが検出された場所のx, y座標に図形を置くことで、「人の動きをなぞるような映像」が生まれます。

さらに、毎フレームごとに画面を少しだけ暗くすることで、図形が徐々に消えていく演出も作っています。

ctx.fillStyle = 'rgba(0,0,0,0.06)';
ctx.fillRect(0, 0, width, height);

↑ 黒色の透明度6%で毎フレーム塗ることで、前の絵が少しずつ消えていく

音も自動で作られる！
Web Audio API で音を生成する

このアートは映像だけでなく、音も自動生成されます。

ブラウザには Web Audio API という機能があり、プログラムで音を作ることができます。

動きが検出された位置のy座標（上下）で音の高さが変わる

音の高さ（ピッチ）の決め方
画面の上部で動き → 高い音（ピアノの右側のような音）
画面の下部で動き → 低い音（ピアノの左側のような音）

モードごとに音の「波形」も変わります。正弦波はなめらか、矩形波はブザーのような音。

なぜiPhoneを使えるの？
Camo Studio の仕組み

普通のパソコンのWebカメラの代わりに、iPhoneの高画質カメラを使いたい場合、そのままでは使えません。

そこで使うのが Camo Studio というソフトです。

Camo StudioがiPhoneを「普通のWebカメラ」に変換するので、JavaScriptから通常通り使える

JavaScriptのコード（1行）でカメラにアクセスできる
navigator.mediaDevices.getUserMedia({ video: true })

このコードを実行すると「カメラの使用を許可しますか？」とブラウザが確認します。許可すると映像を取得できます。

全部つなげると？
1フレームで起きていること

プログラムは1秒間に約30回、以下の処理を繰り返しています。

①フレーム取得： Camo経由でiPhoneのカメラから現在の映像（1枚の画像）を取り込む
↓
②差分計算： 前のフレームと比べて、色が変わったピクセルの座標をリストアップする
↓
③描画： 動きがあった座標に、現在のモードに応じた図形（円 or 曲線）を描く
↓
④フェード： 画面全体に半透明の黒を重ねて、前の図形を少しずつ消す
↓
⑤音生成： ランダムなタイミングで、動きの位置に応じた高さの音を鳴らす
↓
⑥プロジェクター出力： ブラウザ画面をプロジェクターに全画面表示 → ①に戻る
1分ごとにモードが自動切替されます
Mode1（円・正弦波）→ Mode2（均一円・のこぎり波）→ Mode3（曲線・矩形波）→ Mode1 → …

デザインを自由に変えてみよう！
仕様はそのままで、見た目だけカスタマイズする方法

プログラムには 仕様（スペック） と デザイン の2つの側面があります。

仕様は「何をするか」の取り決め。今回は「画像差分法で動きを検出し、3モードの映像と音を出力する」こと。ここは変えません。

デザインは「どう見せるか」。色・形・大きさ・消えるスピードなど、見た目に関わる部分は自由に変更できます。

❌ 仕様（変えてはいけない）
  • 画像差分法で動きを検出する
  • 3つのモードが1分ごとに切替
  • y座標で音の高さが変わる
  • 3種類の波形で音を生成する
✅ デザイン（変えてよい）
  • 図形の形・サイズ
  • 色のテーマ・透明度
  • 消えるスピード（残像の長さ）
  • ブレンドモード（重なり方）
  • 曲線の太さ・広がり
コードのどこを変えると何が変わる？
📐 図形のサイズを変える（Mode 1）
// 変更前（最小4px〜最大32px） const r = 4 + Math.random() * 28; // 変更後：大きくする（最小10px〜最大70px） const r = 10 + Math.random() * 60; // 変更後：小さく均一にする const r = 3 + Math.random() * 5;

最初の数字＝最小サイズ、足す数字が大きいほど最大サイズが大きくなります

🎨 色のテーマを変える
// レインボー（0〜360度 = すべての色） ctx.fillStyle = `hsl(${Math.random()*360}, 100%, 60%)`; // 暖色のみ（赤〜黄、0〜60度） ctx.fillStyle = `hsl(${Math.random()*60}, 100%, 60%)`; // 寒色のみ（青〜水色、180〜260度） ctx.fillStyle = `hsl(${180+Math.random()*80}, 100%, 60%)`; // 1色に固定する（220度＝青） ctx.fillStyle = `hsl(220, 100%, 60%)`;
⏱ 消えるスピードを変える
// 普通（0.06 → 少し残像が残る） ctx.fillStyle = 'rgba(0,0,0,0.06)'; // 早く消える（0.15 → すぐ消える） ctx.fillStyle = 'rgba(0,0,0,0.15)'; // ゆっくり消える（0.02 → 長い残像） ctx.fillStyle = 'rgba(0,0,0,0.02)';

数値が大きいほど黒が濃く塗られるので早く消える、小さいほど残像が残ります

✨ ブレンドモードで光らせる
// 通常（重なっても色が混ざらない） ctx.globalCompositeOperation = 'source-over'; // 加算合成（色が重なって光のように明るくなる） ctx.globalCompositeOperation = 'lighter'; // スクリーン合成（やわらかく発光する感じ） ctx.globalCompositeOperation = 'screen';
